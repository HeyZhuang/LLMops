# LLMOps Platform 快速上手指南

## 🚀 5分钟快速启动

### 前置条件检查清单
- [ ] Python 3.8+ 已安装
- [ ] Node.js 16+ 已安装
- [ ] Docker Desktop 已安装并运行（可选）
- [ ] PostgreSQL 12+ 已安装（或使用Docker）
- [ ] Redis 6+ 已安装（或使用Docker）

### 一键启动（Windows）

```powershell
# 1. 启动Docker服务（仅Weaviate）
start-local.bat

# 2. 启动后端（新终端）
cd imooc-llmops-api\imooc-llmops-api-master
.venv\Scripts\Activate.ps1
python -m app.http.app

# 3. 启动前端（新终端）
cd imooc-llmops-ui\imooc-llmops-ui-master
npm run dev
```

### 访问地址
- **前端界面**：http://localhost:5173
- **后端API**：http://localhost:5000
- **Weaviate**：http://localhost:8080

## 📋 详细启动步骤

### 步骤1：启动Docker服务

#### 方式A：仅启动Weaviate（推荐开发环境）
```powershell
cd docker\docker
docker-compose -f docker-compose-weaviate-only.yaml up -d
```

#### 方式B：启动完整服务栈
```powershell
cd docker\docker
docker-compose up -d
```

#### 验证服务
```powershell
# 检查容器状态
docker ps

# 验证Weaviate
curl http://localhost:8080/v1/meta
```

### 步骤2：配置后端环境

#### 2.1 创建虚拟环境
```powershell
cd imooc-llmops-api\imooc-llmops-api-master
python -m venv venv
venv\Scripts\Activate.ps1
```

#### 2.2 安装依赖
```powershell
pip install -r requirements.txt
```

#### 2.3 配置环境变量
创建 `.env` 文件：
```env
# 数据库配置
DATABASE_URL=postgresql://postgres:password@localhost:5432/llmops

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379

# Weaviate配置
WEAVIATE_HOST=localhost
WEAVIATE_PORT=8080

# LLM API密钥（至少配置一个）
OPENAI_API_KEY=sk-xxx
# 或
QIANFAN_API_KEY=xxx
QIANFAN_SECRET_KEY=xxx
```

#### 2.4 初始化数据库
```powershell
# 运行数据库迁移
python migrate_db.py

# 创建管理员用户
python create_user.py
```

### 步骤3：启动后端服务

```powershell
python -m app.http.app
```

看到以下信息表示启动成功：
```
 * Running on http://127.0.0.1:5000
```

### 步骤4：配置并启动前端

#### 4.1 安装依赖
```powershell
cd imooc-llmops-ui\imooc-llmops-ui-master
npm install
```

#### 4.2 配置API地址（如需要）
编辑 `src/config/index.ts`：
```typescript
export const API_BASE_URL = 'http://localhost:5000'
```

#### 4.3 启动开发服务器
```powershell
npm run dev
```

## 🔧 常见问题排查

### 问题1：后端启动失败

#### 症状
```
Error: Could not connect to database
```

#### 解决方案
1. 检查PostgreSQL是否运行：
   ```powershell
   # Windows
   Get-Service -Name postgresql*
   ```
2. 检查数据库配置：
   - 验证`.env`中的`DATABASE_URL`
   - 确认数据库已创建
3. 测试数据库连接：
   ```powershell
   python test_db_connection.py
   ```

### 问题2：Weaviate连接失败

#### 症状
```
Error: Failed to connect to Weaviate
```

#### 解决方案
1. 检查Weaviate容器：
   ```powershell
   docker ps | findstr weaviate
   ```
2. 查看Weaviate日志：
   ```powershell
   docker logs llmops-weaviate
   ```
3. 重启Weaviate：
   ```powershell
   docker restart llmops-weaviate
   ```

### 问题3：前端API请求失败

#### 症状
```
CORS error 或 404 Not Found
```

#### 解决方案
1. 检查后端是否运行：
   - 访问 http://localhost:5000/ping
   - 应该返回 `{"code": 200, "message": "pong"}`
2. 检查CORS配置：
   - 确认`http.py`中的CORS配置包含前端地址
3. 检查API地址配置：
   - 确认`src/config/index.ts`中的API地址正确

### 问题4：文档解析失败

#### 症状
```
文档状态一直显示"处理中"
```

#### 解决方案
1. 检查Celery是否运行：
   ```powershell
   # 需要启动Celery Worker
   celery -A app.celery worker --loglevel=info
   ```
2. 检查文件格式：
   - 确认文件格式支持（PDF、Word、Markdown等）
   - 检查文件是否损坏
3. 查看任务日志：
   - 检查Celery Worker的日志输出
