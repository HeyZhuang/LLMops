## Q1: 个人空间 vs 探索栏目（应用广场/插件广场）的区别

### 核心定位

| 维度 | 个人空间 (`/space/`) | 探索栏目 (`/store/`) |
|------|---------------------|---------------------|
| **定位** | 用户的**工作区** | 平台提供的**模板市场** |
| **数据归属** | 当前登录用户创建的内容 | 平台预置，对所有用户公开 |
| **数据来源** | 数据库（按 `account_id` 过滤） | YAML 配置文件 + 代码内置 |

### AI 应用对比

#### 个人空间 — AI应用 (`/space/apps`)

- 展示用户**自己创建**的应用
- 支持完整 CRUD：创建、编辑、复制、删除、发布/取消发布、调试、查看分析数据
- 后端通过 `App.account_id == current_user.id` 过滤
- 关键后端接口：
  - `GET /apps` — 分页获取用户应用
  - `POST /apps` — 创建新应用
  - `POST /apps/<app_id>` — 更新应用
  - `POST /apps/<app_id>/delete` — 删除
  - `POST /apps/<app_id>/copy` — 复制
  - `POST /apps/<app_id>/publish` — 发布
  - `POST /apps/<app_id>/cancel-publish` — 取消发布

#### 探索 — 应用广场 (`/store/apps`)

- 展示平台**预置的 Agent 模板**（如旅行助手、LLM产品经理等）
- **只读浏览** + 唯一操作：「添加到工作区」
- 支持按分类筛选（助手、人力资源、写作、电商营销等）
- 模板来源：`internal/core/builtin_apps/builtin_apps/` 下的 YAML 文件
- 创建者统一显示为 "LLMOps-Platform"
- 关键后端接口：
  - `GET /builtin-apps/categories` — 获取分类
  - `GET /builtin-apps` — 获取所有预置应用
  - `POST /builtin-apps/add-builtin-app-to-space` — 添加到工作区

#### 「添加到工作区」的完整流程

1. 前端调用 `POST /builtin-apps/add-builtin-app-to-space`
2. 后端 `BuiltinAppService.add_builtin_app_to_space()` 读取 YAML 模板
3. 在数据库创建**新的 App 副本**（`account_id` = 当前用户）
4. 同时创建对应的 `AppConfigVersion`（草稿配置）
5. 前端跳转到 `/space/apps/:app_id`，用户可自由编辑

### 插件对比

#### 个人空间 — 插件 (`/space/tools`)

- 展示用户**自定义的 API 工具**
- 基于 OpenAPI 3.0 Schema 定义，每个 Provider 可包含多个工具
- 支持完整 CRUD：创建、编辑、删除、验证 Schema
- 可查看每个工具的参数详情（名称、类型、是否必填）
- 关键后端接口：
  - `GET /api-tools` — 分页获取用户自定义工具
  - `POST /api-tools` — 创建
  - `POST /api-tools/<provider_id>` — 更新
  - `POST /api-tools/<provider_id>/delete` — 删除
  - `POST /api-tools/validate-openapi-schema` — 验证 Schema

#### 探索 — 插件广场 (`/store/tools`)

- 展示平台**内置工具**（DuckDuckGo搜索、天气查询、时间工具、Wikipedia、DALL-E 3、Google Serper 等）
- **只读浏览**，不能编辑或删除
- 支持按分类筛选
- 提供「创建自定义插件」按钮，点击后跳转到 `/space/tools?create_type=tool`
- 关键后端接口：
  - `GET /builtin-tools/categories` — 获取分类
  - `GET /builtin-tools` — 获取所有内置工具

### 关键文件索引

| 区域 | 前端 | 后端 Handler | 后端 Service |
|------|------|-------------|-------------|
| 个人空间-应用 | `views/space/apps/ListView.vue` | `app_handler.py` | `app_service.py` |
| 应用广场 | `views/store/apps/ListView.vue` | `builtin_app_handler.py` | `builtin_app_service.py` |
| 个人空间-插件 | `views/space/tools/ListView.vue` | `api_tool_handler.py` | `api_tool_service.py` |
| 插件广场 | `views/store/tools/ListView.vue` | `builtin_tool_handler.py` | `builtin_tool_service.py` |

### 一句话总结

> **探索栏目是"选模板"**——浏览平台预置的应用和工具，看中了就添加到自己的工作区；**个人空间是"做东西"**——创建、编辑、调试、发布自己的 AI 应用和自定义插件。两者是"发现 → 使用"的关系。

## Q2: 项目中 Agent 的完整体现

### 一、Agent 核心引擎（后端 `internal/core/agent/`）

| 文件 | 说明 |
|------|------|
| `agents/base_agent.py` | **BaseAgent 基类**——基于 LangGraph 的 `CompiledStateGraph`，提供 `invoke()`（阻塞执行）和 `stream()`（流式执行）两种模式 |
| `agents/function_call_agent.py` | **FunctionCallAgent**——适用于支持 Tool Call 的模型（如 GPT-4），LangGraph 4 节点工作流：预设操作 → 长期记忆召回 → LLM 调用（绑定工具） → 工具执行，循环直到无工具调用 |
| `agents/react_agent.py` | **ReACTAgent**——适用于不支持原生工具调用的模型，通过提示词引导 LLM 输出 JSON 格式的工具调用（`` ```json {"name":"...","args":{...}}``` ``），手动解析后执行 |
| `agents/agent_queue_manager.py` | **AgentQueueManager**——基于线程 Queue + Redis 的事件分发系统，支持 10 秒心跳、600 秒超时、外部停止信号 |
| `entities/agent_entity.py` | **AgentConfig / AgentState**——配置（最大迭代数、预设提示、是否启用长期记忆、工具列表、审查配置）和状态（消息、迭代计数、历史记忆） |
| `entities/queue_entity.py` | **AgentThought / QueueEvent**——事件体系，包含思考、消息、动作、检索、记忆召回、结束、异常等事件类型 |

#### FunctionCallAgent 的 LangGraph 工作流

```
preset_operation_node（输入校验 & 关键词过滤）
        ↓
long_term_memory_recall_node（构建含记忆的系统提示词）
        ↓
    llm_node（调用 LLM，绑定工具）
        ↓
  ┌─ 有 tool_calls? ─┐
  │ Yes              │ No
  ↓                  ↓
tools_node          END
（执行工具）
  ↓
llm_node（带工具结果再次调用）
  ... 循环直到无工具调用或达最大迭代数
```

#### ReACTAgent 与 FunctionCallAgent 的区别

| 维度 | FunctionCallAgent | ReACTAgent |
|------|-------------------|------------|
| 适用模型 | 支持 `TOOL_CALL` 特性的模型 | 不支持原生工具调用的模型 |
| 工具调用方式 | LLM 原生返回 `tool_calls` | LLM 输出 JSON 文本，手动解析 |
| 系统提示词 | `AGENT_SYSTEM_PROMPT_TEMPLATE` | `REACT_AGENT_SYSTEM_PROMPT_TEMPLATE`（含工具描述） |
| 解析逻辑 | 直接读取结构化响应 | 判断前 7 字符是否为 `` ```json ``，是则解析工具调用，否则视为文本回复 |

### 二、Agent 选择与调用（后端 Service 层）

| 文件 | Agent 相关逻辑 |
|------|----------------|
| `service/web_app_service.py` | **核心调度**：加载 App 配置 → 组装工具列表 → 根据模型能力自动选择 Agent 类型 → 流式执行并推送 SSE |
| `service/app_config_service.py` | 加载应用草稿/发布配置（模型、工具、数据集、工作流绑定） |
| `service/conversation_service.py` | 管理会话历史，Agent 执行后持久化 `MessageAgentThought` 记录 |

Agent 类型的自动选择逻辑：

```python
agent_class = FunctionCallAgent if ModelFeature.TOOL_CALL in llm.features else ReACTAgent
```

### 三、Agent 可用的能力模块

| 能力 | 文件位置 | 说明 |
|------|----------|------|
| **内置工具** | `core/tools/builtin_tools/` | DuckDuckGo 搜索、DALL-E3 画图、高德天气、当前时间、Wikipedia、Google Serper |
| **自定义 API 工具** | `core/tools/api_tools/` | 用户通过 OpenAPI 3.0 Schema 动态生成的工具 |
| **知识库检索** | `core/retrievers/` | 混合检索：语义检索（Weaviate 向量）+ 关键字检索（jieba 分词）+ Cross-Encoder 重排序，作为特殊工具注入 Agent |
| **工作流工具** | `core/workflow/workflow.py` | 已发布的工作流被包装为 `BaseTool`，Agent 可直接调用 |
| **短期记忆** | `core/memory/token_buffer_memory.py` | 按 Token 数裁剪的近期对话历史 |
| **长期记忆** | Agent 节点 `long_term_memory_recall_node` | 对话摘要注入系统提示词，跨会话保持上下文 |
| **内容审查** | Agent 节点 `preset_operation_node` | 输入关键词过滤 + 输出敏感词替换 |

### 四、Agent 流式输出协议（SSE）

| 事件类型 | 含义 |
|----------|------|
| `long_term_memory_recall` | 长期记忆召回 |
| `agent_thought` | Agent 思考过程（选择工具的推理） |
| `agent_action` | 工具调用执行（tool name + tool_input） |
| `dataset_retrieval` | 知识库检索命中结果 |
| `agent_message` | 最终文本回复（累积式流式输出） |
| `agent_end` | 执行完毕 |
| `error` / `timeout` / `ping` | 异常与心跳 |

后端通过 Flask SSE Response（`text/event-stream`）逐事件推送，前端 `ssePost()`（`src/utils/request.ts`）逐行解析。

### 五、Agent 前端配置 UI（`views/space/apps/`）

| 组件 | Agent 配置项 |
|------|-------------|
| `ModelConfig.vue` | 选择 LLM 提供商 + 模型（决定使用 FunctionCall 还是 ReACT） |
| `PresetPromptTextarea` | Agent 系统提示词 / 人设指令 |
| `abilities/ToolsAbilityItem.vue` | 为 Agent 绑定内置工具 + API 工具 |
| `abilities/WorkflowsAbilityItem.vue` | 为 Agent 绑定工作流（最多 5 个） |
| `abilities/DatasetsAbilityItem.vue` | 为 Agent 绑定知识库 + 配置检索策略（top_k、score 阈值） |
| `abilities/LongTermMemoryAbilityItem.vue` | 开关长期记忆 |
| `abilities/OpeningAbilityItem.vue` | 开场白 + 建议问题 |
| `abilities/SuggestedAfterAnswerAbilityItem.vue` | 回答后自动生成追问建议 |
| `abilities/ReviewConfigAbilityItem.vue` | 内容安全审查配置 |

### 六、Agent 对话界面（前端）

| 界面 | 文件 | 场景 |
|------|------|------|
| **调试对话** | `views/space/apps/components/PreviewDebugChat.vue` | 开发者在编辑应用时右侧实时测试 Agent |
| **已发布 WebApp** | `views/web-apps/IndexView.vue` | 终端用户通过 token 访问已发布应用，与 Agent 对话 |
| **Agent 思维链展示** | `components/AgentThought.vue` | 可折叠手风琴展示每步推理：记忆召回 → 思考 → 工具调用 → 知识检索 → 最终回复，附带每步耗时 |
| **AI 消息渲染** | `components/AiMessage.vue` | 整合 AgentThought + Markdown 渲染回复 + Token 统计 + 建议追问 |

前端 SSE 事件处理逻辑（`PreviewDebugChat.vue`）：
- `agent_message` 事件：**累积模式**，文本逐字追加到已有 thought
- 其他事件：**追加模式**，每个事件作为独立推理步骤加入 `agent_thoughts` 数组
- 支持中途停止（调用 `stopDebugChat(app_id, task_id)`）

### 七、Agent 数据持久化

| 模型 | 说明 |
|------|------|
| `App` + `AppConfigVersion` | 存储 Agent 配置（模型、工具、知识库、提示词等） |
| `Conversation` | 对话会话 |
| `Message` | 每轮问答（用户 query + Agent answer + Token 统计） |
| `MessageAgentThought` | **逐步记录** Agent 每次推理的思考、工具调用、检索结果、耗时、Token 消耗 |

Agent 执行完毕后，在后台线程中将所有 `AgentThought` 事件批量写入 `MessageAgentThought` 表，不阻塞 SSE 响应流。

### 八、预置 Agent 模板（应用广场）

| 文件位置 | 说明 |
|----------|------|
| `core/builtin_apps/builtin_apps/*.yaml` | 预置 Agent 模板（旅行助手、LLM 产品经理等），定义好提示词、模型、工具配置 |
| `core/builtin_apps/categories/categories.yaml` | 模板分类（助手、HR、写作、电商营销等） |
| `service/builtin_app_service.py` | 「添加到工作区」时复制模板为用户自己的 App |

### 总结

> 该项目的 **Agent 不是一个独立功能模块，而是整个平台的核心运行机制**。每一个 AI 应用本质上就是一个 Agent 实例——它的运行引擎（LangGraph 状态图）、推理策略（FunctionCall / ReACT）、可用工具（内置/API/工作流/知识库）、记忆系统（短期/长期）、流式输出协议（SSE 事件链）、配置界面、调试界面、以及持久化记录，构成了一条完整的 Agent 生命周期链路。
