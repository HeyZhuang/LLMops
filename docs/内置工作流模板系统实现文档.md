# 内置工作流模板系统实现文档

## 功能概述

为 LLMOps 平台新增**内置工作流模板系统**，用户可以从预设模板一键创建工作流，无需从零搭建。

- `/space/workflows` 页面顶部展示精选推荐模板（前6个）
- `/store/workflows`（工作流广场）页面展示全部模板，支持分类筛选和搜索
- 侧边栏"探索"区新增"工作流广场"导航入口
- 提供 10 个多品类模板，覆盖内容生成、数据处理、客服对话、自动化四大场景

---

## 架构设计

完全复用 `builtin-app`（应用广场）的架构模式：

```
YAML模板文件 → Manager(@singleton加载) → Service(业务逻辑) → Handler(API接口) → 前端展示
```

### 数据流

```
前端请求 → /builtin-workflows/* → BuiltinWorkflowHandler
    → BuiltinWorkflowService → BuiltinWorkflowManager(内存中的YAML数据)
    → 添加到空间时：创建 Workflow 数据库记录（draft_graph 来自模板）
```

| 方法 | 路径 | 说明 |

## 10 个工作流模板详情

### 内容生成（content）

| 模板 | tool_call_name | 节点链路 | 功能说明 |
|------|---------------|---------|---------|
| 文章生成器 | `article_generator` | Start → LLM(生成大纲) → LLM(撰写正文) → End | 输入主题和关键词，先生成大纲再撰写完整文章 |
| 多语言翻译 | `translator` | Start → LLM(翻译文本) → End | 输入文本和目标语言，输出翻译结果 |
| SEO内容优化 | `seo_optimizer` | Start → LLM(SEO分析) → LLM(内容优化) → End | 先分析SEO表现再输出优化版本 |

### 数据处理（data）

| 模板 | tool_call_name | 节点链路 | 功能说明 |
|------|---------------|---------|---------|
| 信息提取器 | `data_extractor` | Start → LLM(提取) → TemplateTransform(格式化) → End | 从非结构化文本提取指定字段，JSON格式输出 |
| JSON格式转换 | `format_converter` | Start → Code(解析转换) → TemplateTransform(格式化) → End | 支持JSON转CSV/Markdown表格/YAML |

### 客服对话（service）

| 模板 | tool_call_name | 节点链路 | 功能说明 |
|------|---------------|---------|---------|
| 智能客服 | `customer_service` | Start → LLM(意图识别) → LLM(生成回复) → End | 先识别意图和情绪，再生成专业客服回复 |
| 知识库问答 | `knowledge_qa` | Start → LLM(问题改写) → LLM(生成答案) → End | 先优化问题表述，再生成详细回答 |

### 自动化（automation）

| 模板 | tool_call_name | 节点链路 | 功能说明 |
|------|---------------|---------|---------|
| 邮件助手 | `email_assistant` | Start → LLM(邮件分析) → LLM(生成回复) → End | 分析邮件意图和关键信息，自动生成回复 |
| 代码审查 | `code_reviewer` | Start → LLM(代码审查) → TemplateTransform(生成报告) → End | 多维度代码审查，输出结构化报告 |
| 周报生成 | `report_generator` | Start → LLM(整理归类) → TemplateTransform(生成周报) → End | 将工作记录分类整理成结构化周报 |

## YAML 模板编写规范

如需新增模板，在 `internal/core/builtin_workflows/builtin_workflows/` 下新建 `.yaml` 文件，Manager 会自动加载。

### 必要字段

```yaml
id: "有效的UUID"          # 全局唯一
category: "content"       # 对应 categories.yaml 中的 category 值
name: "模板名称"
tool_call_name: "英文标识"  # 添加到空间后的工作流名，需全局唯一
icon: ""                  # 图标URL，可为空
description: "模板描述"
author: "LLMOps-Platform"
created_at: 1710720000    # Unix 时间戳
draft_graph:
  nodes: [...]
  edges: [...]
```

### 节点规范

- 所有节点 `id` 必须为有效 UUID，且全局唯一
- 节点 `title` 必须唯一（同一工作流内）
- 变量名 `name` 必须匹配 `^[A-Za-z_][A-Za-z0-9_]*$`
- 可用节点类型：`start`, `llm`, `code`, `template_transform`, `end`
- **不使用** `tool`, `dataset_retrieval`, `http_request` 节点（需要用户配置具体工具ID/知识库ID）

### LLM 节点配置

```yaml
- id: "uuid"
  node_type: "llm"
  title: "唯一标题"
  prompt: "提示词，用 {{变量名}} 引用输入"
  model_config:
    provider: "openai"
    model: "gpt-4o-mini"
    parameters:
      temperature: 0.7
      top_p: 0.9
      frequency_penalty: 0
      presence_penalty: 0
      max_tokens: 4096
  inputs:
    - name: "变量名"
      type: "string"
      required: true
      value:
        type: "ref"          # ref=引用其他节点输出, literal=直接值
        content:
          ref_node_id: "源节点UUID"
          ref_var_name: "output"
  outputs:
    - name: "output"         # LLM输出固定为 output
      value:
        type: "generated"
```

### 边的连接

```yaml
edges:
  - id: "edge-uuid"
    source: "源节点UUID"
    source_type: "start"     # 与源节点的 node_type 一致
    target: "目标节点UUID"
    target_type: "llm"       # 与目标节点的 node_type 一致
```

## 验证步骤

1. **后端启动**：`python -m app.http.app`，访问以下API确认返回正确数据
   - `GET /builtin-workflows/categories` → 4 个分类
   - `GET /builtin-workflows` → 10 个模板
   - `POST /builtin-workflows/add-builtin-workflow-to-space` → 创建工作流

2. **前端启动**：`npm run dev`
   - 访问 `/store/workflows` → 工作流广场页面展示全部模板
   - 访问 `/space/workflows` → 顶部推荐模板区域
   - 点击"添加到工作区" → 创建成功并跳转到工作流编辑页
   - 编辑页中 DAG 图正确渲染节点和边

3. **类型检查**：`npm run type-check` → 零错误通过

## 扩展指南

### 新增模板

1. 在 `builtin_workflows/` 目录下新建 YAML 文件
2. 按上述规范填写节点和边
3. 重启后端服务即可自动加载

### 新增分类

1. 编辑 `categories/categories.yaml`，添加新分类
2. 重启后端服务即可

### 前端自定义

- 工作流广场页面：`src/views/store/workflows/ListView.vue`
- 推荐模板数量：修改 `ListView.vue` 中 `builtinWorkflows.value.slice(0, 6)` 的数字
- 侧边栏链接：`src/views/layouts/components/LayoutSidebar.vue`
