                                                                                                           
  三种检索策略                                                                                                
  1. 语义检索（SemanticRetriever）                                                                         
                                                                                                           
  技术：Embedding 向量 + Weaviate ANN 近似最近邻搜索

  Query "如何部署模型"
    → HuggingFace API 转为 1024维向量
    → Weaviate 余弦相似度搜索（HNSW 索引）
    → 按 dataset_id / document_enabled / segment_enabled 过滤
    → 返回 Top-K 片段 + 相似度得分

  核心代码在 semantic_retriever.py:33：
  vector_store.similarity_search_with_relevance_scores(
      query=query, k=k,
      filters=Filter.all_of([
          Filter.by_property("dataset_id").contains_any([...]),
          Filter.by_property("document_enabled").equal(True),
          Filter.by_property("segment_enabled").equal(True),
      ])
  )

  优势：理解语义，"部署模型"能匹配到"模型上线步骤"
  劣势：可能丢失精确关键词（如代码片段、专有名词）

  2. 全文检索（FullTextRetriever / BM25）

  技术：Jieba 中文分词 + 关键词倒排索引 + BM25 算法

  Query "如何部署模型"
    → Jieba 分词: ["部署", "模型"]
    → 查 PostgreSQL 关键词倒排索引表（KeywordTable）
    → BM25 算法计算词频/逆文档频率得分
    → 返回 Top-K 片段

  核心流程：
  - 索引阶段：jieba_service.extract_keywords() 提取每个片段的关键词，存入 KeywordTable（JSONB
  格式的倒排索引）
  - 检索阶段：对 query 分词 → 从倒排索引找候选片段 → rank-bm25 库排序

  优势：精确关键词匹配，对专有名词、代码片段效果好
  劣势：不理解语义，"部署"匹配不到"上线"

  3. 混合检索（EnsembleRetriever）

  技术：LangChain EnsembleRetriever 加权融合

  hybrid_retriever = EnsembleRetriever(
      retrievers=[semantic_retriever, full_text_retriever],
      weights=[0.5, 0.5],  # 各占50%权重
  )

  两路结果按权重合并去重，取长补短。

  策略选择逻辑

  retrieval_service.py:82：
  if retrieval_strategy == "semantic":
      lc_documents = semantic_retriever.invoke(query)[:k]
  elif retrieval_strategy == "full_text":
      lc_documents = full_text_retriever.invoke(query)[:k]
  else:  # hybrid
      lc_documents = hybrid_retriever.invoke(query)[:k]

  Embedding 缓存层

  CacheBackedEmbeddings → Redis（hash为key） → HuggingFace API

  相同文本不重复调用 API，命中缓存直接返回向量。

  一句话总结

  语义检索负责"理解意思"，全文检索负责"精确匹配"，混合检索 5:5 加权融合两者结果，Embedding 经 Redis        
  缓存避免重复调用，最终返回 Top-K 相关片段注入 LLM 上下文。
