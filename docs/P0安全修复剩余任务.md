## 剩余任务

### 1. 安装 RestrictedPython 依赖

```bash
cd imooc-llmops-api/imooc-llmops-api-master
# 激活虚拟环境后执行
pip install RestrictedPython==7.4
```

### 2. 配置环境变量

在 `.env` 文件中补充以下配置：

```env
# Debug 模式（开发环境设为 1，生产环境设为 0 或不设置）
FLASK_DEBUG=1

# Poppler 路径（Windows PDF 解析依赖）
POPPLER_PATH=D:\dev\poppler\poppler-24.08.0\Library\bin
```

### 3. 验证修复效果

#### 3.1 代码沙箱验证

在工作流 CodeNode 中尝试执行以下恶意代码，应被拦截：

```python
def main(params):
    import os
    os.system("whoami")
    return {"result": "hacked"}
```

预期结果：抛出异常，`__import__` 不在白名单中，代码无法执行。

#### 3.2 认证验证

未登录状态下访问统计分析接口：

```bash
curl http://localhost:5000/api/apps/<app_id>/analysis
```

预期结果：返回 401 Unauthorized。

#### 3.3 Debug 模式验证

不设置 `FLASK_DEBUG` 环境变量启动服务，确认 Flask 以非 debug 模式运行（控制台无 debugger PIN 输出）。

#### 3.4 Print 语句验证

```bash
grep -rn "print(" internal/handler/api_key_handler.py
```

预期结果：无匹配行。

### 4. Weaviate 生产部署配置

生产环境 `docker-compose.yaml` 中已将 `AUTHENTICATION_ANONYMOUS_ACCESS_ENABLED` 设为 `false`。部署前需：

1. 配置 Weaviate API Key 认证，参考：https://weaviate.io/developers/weaviate/configuration/authentication
2. 在后端 `.env` 中配置 Weaviate 认证凭据
3. 更新 `weaviate-client` 连接代码以传入认证信息
