# LLMOps 项目启动指南

## 前置条件
- Docker Desktop 已安装并运行
- Python 虚拟环境已激活

## 启动方式选择

### 🚀 方式一：本地开发模式（推荐）
**适用于：已安装本地 PostgreSQL 和 Redis 的开发者**

使用本地数据库服务，Docker 仅运行 Weaviate 向量数据库，资源占用更少，启动更快。

#### 快速启动
```powershell
# 使用一键启动脚本
start-local.bat

# 或手动启动
cd docker/docker
docker-compose -f docker-compose-local.yaml up -d
```

#### 停止服务
```powershell
# 使用一键停止脚本
stop-local.bat
```

### 🐳 方式二：完整 Docker 服务栈
**适用于：希望所有服务都运行在 Docker 中**

#### 启动完整服务栈
```powershell
# 进入 Docker 配置目录
cd docker/docker

# 启动所有服务（包括 PostgreSQL、Redis、Weaviate）
docker-compose up -d

# 查看服务状态
docker-compose ps
```

#### 仅启动 Weaviate
```powershell
# 进入 Docker 配置目录
cd docker/docker

# 仅启动 Weaviate 服务
docker-compose -f docker-compose-weaviate-only.yaml up -d

# 查看 Weaviate 状态
docker-compose -f docker-compose-weaviate-only.yaml ps
```

## 2. 验证 Weaviate 服务

### 检查容器状态
```powershell
# 查看 Weaviate 容器日志
docker logs llmops-weaviate --tail 20

# 验证端口是否正确暴露
docker port llmops-weaviate
```

### 验证服务可用性
```powershell
# 测试 HTTP 端口 (8080)
curl http://localhost:8080/v1/meta

# 或在浏览器中访问
# http://localhost:8080/v1/meta
```

## 3. 启动后端应用

### 激活虚拟环境并启动
```powershell
# 进入后端项目目录
cd imooc-llmops-api/imooc-llmops-api-master

# 激活虚拟环境（如果尚未激活）
.venv\Scripts\Activate.ps1

# 启动后端应用
python -m app.http.app
```

## 4. 验证后端连接

### 检查后端日志
启动后端时，应该看到类似以下的成功信息：
- 没有 Weaviate gRPC 连接错误
- 应用正常启动并监听端口

### 测试 Weaviate 连接
```powershell
# 在后端目录中测试连接
cd imooc-llmops-api/imooc-llmops-api-master
python -c "from internal.core.language_model.language_model_manager import LanguageModelManager; lmm = LanguageModelManager(); print('✅ 后端连接 Weaviate 成功')"
```

## 5. 停止服务

### 停止后端应用
```
Ctrl + C
```

### 停止 Docker 服务
```powershell
# 停止完整服务栈
cd docker/docker
docker-compose down

# 或停止仅 Weaviate 服务
docker-compose -f docker-compose-weaviate-only.yaml down
```

## 6. 故障排除

### 如果 Weaviate 启动失败
```powershell
# 查看详细日志
docker logs llmops-weaviate

# 重启 Weaviate 容器
docker restart llmops-weaviate
```

### 如果后端连接失败
```powershell
# 检查 Weaviate 是否正在运行
docker ps | findstr weaviate

# 检查端口是否被占用
netstat -an | findstr :8080
netstat -an | findstr :50051
```

## 7. 重要端口信息

- **Weaviate HTTP**: `localhost:8080`
- **Weaviate gRPC**: `localhost:50051`
- **PostgreSQL**: `localhost:5432`
- **Redis**: `localhost:6379`
- **后端应用**: 通常在 `localhost:5000` 或其他配置的端口

## 8. 快速启动脚本

您可以创建一个批处理文件来快速启动：

### 创建 `start.bat`
```batch
@echo off
echo 启动 LLMOps 服务...

echo 1. 启动 Docker 服务
cd docker\docker
docker-compose up -d

echo 2. 等待服务启动...
timeout /t 10

echo 3. 启动后端应用
cd ..\..\imooc-llmops-api\imooc-llmops-api-master
call .venv\Scripts\Activate.bat
python -m app.http.app

pause
```

### 创建 `stop.bat`
```batch
@echo off
echo 停止 LLMOps 服务...

cd docker\docker
docker-compose down

echo 服务已停止
pause
```

## 配置说明

### Weaviate 配置
- **版本**: 1.23.7
- **HTTP 端口**: 8080
- **gRPC 端口**: 50051
- **跳过健康检查**: 已启用 (`skip_init_checks=True`)

### 环境变量
确保 `.env` 文件中包含：
```
WEAVIATE_HOST=localhost
WEAVIATE_PORT=8080
```

现在您可以使用这些命令轻松启动和管理您的 LLMOps 项目！
