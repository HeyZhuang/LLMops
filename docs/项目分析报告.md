## 📖 项目概述

**LLMOps Platform** 是一个**企业级大语言模型应用开发与运维平台**，提供从AI应用创建、知识库管理、工作流编排到API集成的全链路解决方案。

### 项目定位
- **目标用户**：企业开发者、AI应用构建者
- **核心价值**：降低LLM应用开发门槛，提供生产级LLMOps解决方案
- **技术特点**：微服务架构、插件化设计、多模型支持

## 🏗️ 技术架构亮点

### 1. 分层架构设计

```
Handler层 (控制器)
    ↓
Service层 (业务逻辑)
    ↓
Core层 (核心能力)
    ↓
Model层 (数据模型)
```

**优势**：
- 职责清晰，易于维护
- 支持依赖注入，便于测试
- 符合SOLID原则

### 2. 依赖注入（Dependency Injection）

使用 `injector` 库实现依赖注入：

```python
@inject
@dataclass
class ConversationService(BaseService):
    db: SQLAlchemy
    language_model_manager: LanguageModelManager
```

**优势**：
- 降低耦合度
- 便于单元测试
- 支持单例模式

### 3. 策略模式（Strategy Pattern）

LLM提供商采用策略模式：

```python
class LanguageModelManager:
    def get_model(self, provider: str, model_name: str):
        strategy = self._get_strategy(provider)
        return strategy.create_model(model_name)
```

**优势**：
- 易于扩展新的LLM提供商
- 统一接口，便于切换

### 4. 异步任务处理

使用 **Celery** 处理耗时操作：

- 文档解析
- 向量化处理
- 批量数据处理

**优势**：
- 提升用户体验
- 支持分布式处理
- 任务状态跟踪

### 5. 多数据库支持

- **PostgreSQL**：关系型数据存储
- **Redis**：缓存和会话存储
- **Weaviate/FAISS**：向量数据库
- **COS**：对象存储（文件存储）

## 🚀 快速上手指南

### 一、环境准备

#### 1. 系统要求
- **Python**: 3.8+
- **Node.js**: 16+
- **PostgreSQL**: 12+
- **Redis**: 6+
- **Docker**: 20.10+ (可选)

#### 2. 克隆项目
```bash
git clone <repository-url>
cd LLMops
```

### 二、后端启动

#### 1. 安装依赖
```bash
cd imooc-llmops-api/imooc-llmops-api-master
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

#### 2. 配置环境变量
创建 `.env` 文件：
```env
# 数据库配置
DATABASE_URL=postgresql://user:password@localhost:5432/llmops

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379

# Weaviate配置
WEAVIATE_HOST=localhost
WEAVIATE_PORT=8080

# LLM API密钥
OPENAI_API_KEY=your_openai_key
QIANFAN_API_KEY=your_qianfan_key
QIANFAN_SECRET_KEY=your_qianfan_secret
```

#### 3. 初始化数据库
```bash
# 运行数据库迁移
python migrate_db.py

# 创建管理员用户
python create_user.py
```

#### 4. 启动Docker服务（可选）
```bash
# 方式一：仅启动Weaviate
cd docker/docker
docker-compose -f docker-compose-weaviate-only.yaml up -d

# 方式二：启动完整服务栈
docker-compose up -d
```

#### 5. 启动后端服务
```bash
python -m app.http.app
```

后端服务默认运行在 `http://localhost:5000`

### 三、前端启动

#### 1. 安装依赖
```bash
cd imooc-llmops-ui/imooc-llmops-ui-master
npm install  # 或 yarn install
```

#### 2. 配置API地址
编辑 `src/config/index.ts`：
```typescript
export const API_BASE_URL = 'http://localhost:5000'
```

#### 3. 启动开发服务器
```bash
npm run dev  # 或 yarn dev
```

前端服务默认运行在 `http://localhost:5173`

#### 1. 登录系统

### 五、常见问题

#### Q1: Weaviate连接失败
**解决方案**：
```bash
# 检查Weaviate是否运行
docker ps | grep weaviate

# 检查端口是否被占用
netstat -an | grep 8080
```

#### Q2: 数据库连接失败
**解决方案**：
- 检查PostgreSQL是否运行
- 验证`.env`中的数据库配置
- 确认数据库已创建

#### Q3: 前端API请求失败
**解决方案**：
- 检查后端服务是否运行
- 验证CORS配置
- 检查API地址配置
